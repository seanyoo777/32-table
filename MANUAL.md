## 1. 시스템 전체 구조

```
홈 (/)
├── 랭킹 관리 (/rankings)       선수·페어·단체팀 CRUD, CSV 가져오기/내보내기, 전적 조회
├── 대회·대진표 (/tournament)   토너먼트·리그·조별전, 결과입력, 부전승, CSV 내보내기
├── 경기일정표 (/schedule)      자동 코트 배치, 다일차 지원, CSV/인쇄
├── 점수 입력 (/score)          실시간 스코어보드 + 직접입력, 검증
├── 운영 대시보드 (/dashboard)  LIVE 현황, 경기 호출(콜링), 미확인 기록 관리
├── 체크인 (/checkin)           QR 스캔, 수동 체크인, 출석 CSV
├── 라이브 보드 (/liveboard)    TV/프로젝터 전광판 (전체화면)
└── 설정 (/settings)           대회 정보, JSON 백업/복원, 데이터 초기화
```

## 2. 대회 전 준비 체크리스트

### 2-1. 초기 설정 (설정 페이지)
1. **설정** → 주최기관명·담당자·시즌·연락처 입력 → **저장**
2. 이전 대회 데이터가 있으면 **JSON 파일 복원** (백업 파일 업로드)

### 2-2. 선수 등록 (랭킹 관리)
| 방법 | 경로 |
|------|------|
| 개별 등록 | 랭킹 → 선수 등록 버튼 → 이름·학교·부문·성별·포인트·등록번호·연락처 입력 |
| CSV 일괄 등록 | 랭킹 → CSV 가져오기 → 양식 다운로드 → 작성 후 업로드 → 중복 자동 감지 |
| 기존 데이터 복원 | 설정 → JSON 파일 복원 |

**CSV 양식**: `이름,학교,부문,성별,포인트`  
부문: `초등 / 중등 / 고등 / 대학 / 일반 / 생활체육`  
성별: `남 / 여`

### 2-3. 복식 페어 등록
랭킹 → 복식 탭 → 페어 등록 → 선수1·선수2 선택 → 종류(남복/여복/혼복) 선택

### 2-4. 단체팀 등록
랭킹 → 단체팀 탭 → 팀 등록 → 팀명·소속·부문·성별·구성원 선택

## 3. 대회 생성

**대회·대진표 → 새 대회 생성**

| 항목 | 설명 |
|------|------|
| 대회명 | 예: 2026 전국 초중고 탁구 선수권대회 |
| 날짜 / 장소 | 대회일·경기장명 |
| 대회 등급 | S급 / A급 / B급 / C급 / 생활체육S~B (포인트 배율에 영향) |
| 경기 방식 | 5세트제 11점 (기본) / 3세트제 / 7세트제 |

### 종목 추가
각 종목마다 → 부문·종류·성별·대진형식 선택

| 대진형식 | 적합 인원 | 특징 |
|---------|----------|------|
| 토너먼트 | 8~128명 | 단판 탈락, 시드 배정 |
| 리그 | 4~12명 | 전원 대전, 승점 순위 |
| 조별+토너먼트 | 16~64명 | 예선 조별 → 본선 토너먼트 |

→ **대회 생성** 클릭 → 각 종목에서 **참가자 배정** → **대진표 생성**

## 4. 경기일정표 생성

**경기일정표 → 새 일정표 만들기**

1. 일정표 이름·날짜·시작시간 입력
2. 코트 수·경기당 시간(분)·버퍼 설정
3. **종목별 세부 설정** (경기 수·분·코트 수 자동 계산)
4. 다일차 대회: **일차 추가** 버튼
5. **일정표 생성** → 코트순·시간순 보기 → **CSV 내보내기** or **인쇄**

### 대회 경기에 시간 배정
일정표 상세 → **대회 경기 배정** → 대회 선택 → **배정** → 대진표에 시간·코트 자동 기재

## 5. 대회 당일 운영

### 5-1. 체크인 (선수 입장 확인)
**체크인 → 체크인 스테이션**

- **QR 스캔**: QR 리더기로 선수증 스캔 (USB HID 방식, 별도 드라이버 불필요)
- **수동 검색**: 이름/소속/등록번호 검색 → 체크인 버튼
- 현황 탭: 부문별 체크인 진행률 확인
- QR 선수증 출력: QR 선수증 탭 → 인쇄

> **QR 선수증 사전 준비**: 대회 전날 QR 선수증 출력 → 선수에게 배포

### 5-2. 대시보드 열기
**운영 대시보드** → 실시간 시계 + 현황 표시  
→ 브라우저 알림 허용 (우상단 **알림 허용** 버튼 클릭)

### 5-3. 경기 호출 (콜링 시스템)
대시보드 → **경기 호출** 섹션
1. 대회 선택 → 종목 선택 → 경기 선택
2. 탁구대 번호 입력 → **호출** 클릭
3. → 라이브 보드에 즉시 표시 + 브라우저 알림 전송
4. 선수 입장 확인 후 → **확인** 클릭

### 5-4. 실시간 스코어 입력 (모바일 최적화)
**점수 입력 → 실시간 스코어** (주심 스마트폰 사용 권장)

1. 대회·종목·경기 선택 → 탁구대 번호 → **스코어보드 시작**
2. 파란(선수1)/빨간(선수2) 버튼 터치 → 점수 자동 집계
3. 서브 표시 자동 (2점마다 교체, 듀스 시 1점마다)
4. 세트 완료 시 자동 다음 세트 이동
5. 경기 종료 → 포인트/Elo 자동 반영 → 대진표 자동 업데이트

**취소(Undo)**: 점수 아래 ↩ 버튼

**키보드 단축키** (스코어보드 화면):
| 키 | 동작 |
|----|------|
| `[` | 선수1 점수 +1 |
| `]` | 선수2 점수 +1 |
| `z` | 선수1 마지막 점수 취소 |
| `x` | 선수2 마지막 점수 취소 |

### 5-5. 수동 점수 입력
**점수 입력 → 직접입력**

1. 대회·종목·경기 선택
2. 세트별 점수 입력 (Enter로 세트 추가)
3. 결과 미리보기 확인 → **결과 저장 & 포인트 반영**
4. 미확인 상태로 저장 → 대시보드에서 확인 처리

### 5-6. 라이브 보드 (TV/프로젝터)
**라이브 보드** 페이지 → **⛶ 전체화면** 클릭  
경기장 모니터/빔프로젝터에 띄워두면 자동 업데이트:
- 현재 진행중 경기 (실시간 점수)
- 경기 호출 목록
- 대기중 경기
- 최근 결과
- 종목별 진행률

## 7. 랭킹 관리

### 포인트 시스템
- 토너먼트 종료 시 승자→`pointsForWin`, 패자→`pointsForWin × 20%` 자동 적립
- **수동 조정**: 선수 행 → **+P** 버튼 → 포인트 입력 → 승/패 체크

### Elo 레이팅 (USATT 방식)
- 초기 레이팅: 포인트 기반 자동 환산 (1000~2800)
- 경기마다 자동 갱신 (K-factor: 초보 32, 중급 24, 고수 16)
- 등급 표시: Beginner / Developing / Intermediate / Advanced / Expert / Elite

### 선수 전적 조회
랭킹 → 선수 행 → 📊 아이콘 → 대회 경기 이력 + 점수 기록 + 승률 확인

### 내보내기
| 버튼 | 내용 |
|------|------|
| CSV 내보내기 | 이름·학교·부문·성별·포인트·Elo·등록번호·연락처 |
| CSV 가져오기 | 양식 기반 일괄 등록 (중복 자동 건너뜀) |

## 12-2. v3.4 추가 기능 (이번 세션)

### 포인트·랭킹 자동 집계 (완전 재구성)
- **등급별 포인트 자동 지급**: 스코어 입력 즉시 `GRADE_POINTS` 테이블 기준 자동 계산 (S급 우승 1000점 ~ C급 참가 2점)
- **종목 배율**: 단식 1.0× / 단체전 0.8× / 복식 0.7× / 혼합복식 0.6× (`getEventMultiplier`)
- **Elo 이변 배율**: 하위 랭킹이 상위를 이기면 최대 1.8×, 예상된 승리는 0.6× (`eloPointsMultiplier`)
- **라운드 성적 자동 부여**: 패자는 탈락 시 즉시 해당 라운드 포인트 수령 (8강 탈락 → 8강 포인트)
- **참가 포인트**: 이벤트 첫 경기 결과 시 모든 참가자 자동 지급 (중복 방지 플래그 `participationAwarded`)
- **복식·단체전 분배**: pair ID → 두 선수, team ID → 전 팀원에게 포인트 분배
- **Elo 레이팅 자동 업데이트**: 단식만 (`recordMatchResult` 내부 처리 — Score.tsx·Tournament.tsx 중복 코드 제거)

> **구현 메모**: 포인트 계산 로직을 `useStore.ts recordMatchResult` 내부로 통합. 어느 화면에서 점수 입력해도 동일하게 적용. `ratingUtils.ts`에 `getEventMultiplier`, `eloPointsMultiplier` 추가.

### 시드예선 대진 방식 신규 추가
- **BracketFormat에 `'시드예선'` 추가**: 토너먼트·리그·조별+토너먼트에 이어 4번째 방식
- **상위 N명 본선 직행**: 시드 수(2/4/8/16/32) 선택, 랭킹 상위 N명은 본선 브래킷 직배정
- **나머지는 예선 조별 미니 토너먼트**: 예선 조 우승자가 본선 슬롯 자동 점령 (`wireSeededQualWinners`)
- **브래킷 뷰**: 예선/본선 라운드 구분 표시 (`phase: 'qual' | 'main'`)
- **경기일정 연동**: `SmartBracketFormat`에 `'seeded'` 추가 → 예선+본선 경기 수 자동 계산
- **시드 수 UI**: 경기일정 자동 입력 화면에서 시드예선 선택 시 "시드 N명" 드롭다운 표시

### 대회 생성 폼 UX 개선
- **스티키 단계 네비게이션**: 스크롤해도 상단 고정 — ① 기본정보 / ② 종목구성 빠른 이동 버튼
- **상단 "✓ 대회 생성" 버튼**: 길게 스크롤 없이 상단에서 바로 저장 가능

### 경기일정표 (대폭 개선)
- **종목별 경기시간 차등**: 개인전 30/25분 vs **단체전 120분(2시간)** 별도 설정
- **단체전 전용 코트 분리**: "전용 코트 N개" 지정 → 개인·복식과 코트 자동 분리 (시간 겹침 방지)
- **부문별 참가/비참가 체크박스**: 대회에 없는 부문 제외 (행 비활성화 + 집계·생성 제외)
- **예상 소요시간 표시**: `필요 작업량 ÷ 코트 수` → 코트 늘리면 소요시간 감소 (4코트 9h → 8코트 4.5h)
- **부문별 진행 일차 범위**: 예선→앞날, 준결승/결승→마지막날 자동 분산
- **⚡ 일괄입력 행**: 종목 열별로 전 부문 인원 한번에 채우기
- 시간 입력칸 가독성 개선 (오전/오후 컬러 뱃지 하단 배치)

> **구현 메모**: 경기일정 수용량 모델은 "코트-분(코트수×운영시간)" 기반. 단체전 등 혼합 경기시간 정확 반영.
> 예상 소요시간 = `필요 코트-분 × 총 운영시간 ÷ 총 코트-분`. 핵심 함수: `scheduleUtils.ts`의
> `matchMinutes`, `calcDayCourtMinutes`, `calcDayOperatingMinutes`, `previewSmartPlan`, `generateSmartSlots`.
